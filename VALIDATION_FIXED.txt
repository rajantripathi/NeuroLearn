📖 CONTEXT: WHY TWO DIFFERENT HOSTS?

Docker Network (inside containers):
  - Containers talk to each other using service names
  - "ollama" resolves to the ollama container
  - Used by: neurolearn-app container
  - Config: OLLAMA_HOST = "http://ollama:11434"

Host Network (your terminal):
  - Can't resolve "ollama" hostname
  - Needs to use "localhost" since Ollama port is exposed
  - Used by: validate_config.py when run from host
  - Auto-detected: VALIDATION_OLLAMA_HOST = "http://localhost:11434"

Both point to the same Ollama service!
